# Juris-Mで歴史学

[jss-lewis-sato.csl](jss-lewis-sato.csl)　[jss-history.csl](jss-history.csl)

ZoteroおよびJuris-Mで使用できるCSL-Mスタイルファイルです。

- **jss-lewis-sato.csl** — 『社会学評論』スタイルガイドに準拠した引用・文献一覧スタイルです。
- **jss-history.csl** — 歴史学の日本語論文向けスタイルです。日本語文献を以下の形式で出力します。

| 文献種別 | Zoteroの文献タイプ | 出力形式 |
|---|---|---|
| 雑誌論文 | Journal Article | 著者、出版年「タイトル」『雑誌名』巻数、ページ数。 |
| 単著・編著 | Book | 著者、出版年『タイトル』出版社。 |
| 論文集所収論文 | Book Section | 著者、出版年「タイトル」編者編『書名』出版社、ページ数。 |
| 学位論文 | Thesis | 著者、出版年『タイトル』大学名博士論文。 |
| 新聞記事 | Newspaper Article | （著者、）年月日「タイトル」『紙名』。 |
| 史料・古文書 | Manuscript | （著者、）年「文書名」所蔵機関名、請求番号。 |

英語文献は標準的なシカゴスタイル（著者―年方式）で出力します。

**ZoteroとJuris-Mのどちらでも使用できます。** ただし以下のCSL-M固有の機能はJuris-Mでのみ利用可能です。

| 機能 | Zotero | Juris-M |
|---|---|---|
| 日英で書式を切り替える | ✓ | ✓ |
| 歴史学向け日本語フォーマット | ✓ | ✓ |
| 「再録：」表記 | ✗ | ✓ |
| 著者名のローマ字読みによるソート | ✗ | ✓ |
| 翻訳タイトル等の併記 | ✗ | ✓ |

### フォーク元からの変更点

フォーク元（keisato0/jss-csl）に含まれる `lewis-original.csl`・`jss-lewis-sato.csl` はそのままです。新たに `jss-history.csl` を追加しました。

`jss-history.csl` は `jss-lewis-sato.csl` をベースに、歴史学の日本語論文向けの文献一覧形式（`history-bibliography-ja` マクロ）を追加したものです。あわせて以下のバグを修正しています。

- 雑誌論文・単著において、出版年とタイトルの間に誤って `、` が挿入される問題
- 雑誌論文において、巻数とページ数の間に `、` が出力されない問題

## はじめに <a name="hajime"></a>

とにかく早く使いたいという方はこのセクションは飛ばしてもらってかまいません。

- CSLファイルとは、Zoteroなど文献管理ソフトから文献情報を出力する際のフォーマットを指定するスタイルファイルです（[公式サイト](https://citationstyles.org/)および[簡単な概要](https://docs.citationstyles.org/en/1.0.1/primer.html)、書き方については[CSL 1.0.1 Specification](https://docs.citationstyles.org/en/1.0.1/specification.html)を参照）。多くの学術雑誌がそれぞれの指定する形式を出力するCSLファイルを開発し公開しています。

- ふつうZoteroと単独のCSLファイルの組み合わせでは複数の言語の文献に異なったスタイル（パンクチュエーションなど）を適用し出力することはできませんが、Zoteroベースの文献管理ソフト[Juris-M](https://juris-m.github.io/)と、CSLから派生した[CSL-M](https://citeproc-js.readthedocs.io/en/latest/csl-m/index.html)の組み合わせでは可能です。この二つはいずれも名古屋大学の[Frank Bennett](https://twitter.com/fgbjr)先生が開発されたものです。

- ただし本リポジトリのCSL-Mファイルは、Juris-M固有の拡張機能（再録表記・ローマ字ソートなど）を除き、**Zoteroでも動作します**。日本語・英語の書式切り替えを含む基本的な機能はZoteroでも利用可能です。

- CSLファイルに比べるとCSL-Mファイルはまだあまり公開されていませんが、そのなかの一つに一橋大学の[Jonathan Lewis](https://www.soc.hit-u.ac.jp/teaching_staff/lewis.html)先生が開発された[「日本社会学会(author-date, Japanese)」](https://github.com/Juris-M/jm-styles/blob/master/jm-japan-sociological-society.csl)があります。

  - これはその名の通り日本社会学会の出している[「社会学評論スタイルガイド」](https://jss-sociology.org/bulletin/guide/)に沿ったCSL-Mスタイルファイルです。「日本社会学会」は（驚くべきことに）Juris-Mのデフォルトで入っている数少ないスタイルの一つです。
  - このリポジトリには lewis-original.cslという名前で入っています。

- lewis-original.cslを修正したものが jss-lewis-sato.csl、それをさらに歴史学向けに改変したものが jss-history.cslです。



## つかいかた <a name="howto"></a>

### Zoteroで使う <a name="zotero"></a>

1. CSLファイルをダウンロードします。
   - このリポジトリの [jss-history.csl](jss-history.csl) のページで `Raw` ボタンを押し、**拡張子を .csl** として保存してください。
2. ZoteroにCSLファイルを読み込みます。
   - Zoteroの `編集`（macOSでは `Zotero` メニュー）>`設定`>`引用`>`スタイル` と進み、右下の `+` ボタンから保存したファイルを追加します。スタイル一覧に `Juris-Mで歴史学（日本語）` と表示されます。
3. 各文献の `言語` フィールドを設定します。
   - 日本語文献：`ja`
   - 英語文献：`en`
   - **この設定がないと日本語文献が英語スタイルで出力されます。**
4. 文献一覧を出力します。
   - 文献を選択して右クリック → `出典表記／参考文献目録を作成...` → 引用スタイルに `Juris-Mで歴史学（日本語）` を選択します。

#### 史料・古文書の入力フィールド

Zoteroで `Manuscript`（手稿）タイプを選択し、以下のフィールドを使用します。

| Zoteroのフィールド | 入力内容 |
|---|---|
| タイトル | 文書名 |
| 著者 | 作成者（不明の場合は空欄） |
| 日付 | 文書の年代 |
| アーカイブ | 所蔵機関名 |
| アーカイブの場所 | 請求番号・整理番号 |

### Juris-Mで使う <a name="jurism"></a>

> **注意：** Juris-Mは2023年リリースのバージョン6（Zotero 6ベース）を最後に更新が停止しています。Zotero 7/8ベースの新バージョンは2026年4月時点で未リリースです。新しいOSでの動作に問題が生じる可能性があります。最新情報は[公式フォーラム](https://forums.zotero.org/discussion/127782/is-juris-m-still-active)を参照してください。

以下の説明はmacOSを日本語設定で使用するユーザ用です。他のOSの方や他の言語で使っている方は適宜読み替えてください。メニュー名はZotero 6ベースのものです（Zoteroとは異なります）。

1. Juris-Mをダウンロードします。[公式サイト](https://juris-m.github.io/release/)の「Jurism Client Installer」をクリックしてください。インストールが終わったら、Juris-Mを立ち上げておいてください。
2. CSLファイルをダウンロードします（手順はZoteroの場合と同様です）。
3. Juris-MにCSLファイルを読み込みます。
   - `環境設定`>`引用`>`引用スタイル` と進み、右下の `＋` ボタンから先ほどのファイルを追加します。スタイル一覧に `Juris-Mで歴史学（日本語）` と表示されます。
4. Juris-Mの言語設定をします。Lewis先生の[チュートリアル](https://juris-m.readthedocs.io/en/latest/tutorial.html)に詳しいですが、必要な作業は以下のとおりです。
   - `環境設定`>`Languages` と進み、`「言語を追加」` フィールドで「English」と「Japanese」を追加してください。
     - ニックネームは自由に変えてください。以下では `Japanese` の表示を、デフォルトの「ja」から「Japanese」に変えています。
   - 「Japanese」の右にある `＋` ボタンを押し、`表現形態等`>`ALA-LC Romanization, 1997 Edition` を追加してください。ローマ字表記を登録できる日本語のサブフィールドです。ニックネームは「Romaji」としておきます。
   - `引用と参考文献` の `ソート` を設定します。文献一覧を著者のローマ字表記で並べ替えたいので、「Romaji」の行のチェックボックスを押してください。同様に `音訳` の列でも、「Romaji」の行と交わるチェックボックスにチェックしてください。
5. 文献を登録します。Juris-MにはZotero同様のブラウザ拡張機能がありますが、対応状況は各ブラウザによって異なります。[公式サイト](https://juris-m.github.io/release/)を確認してください。手入力ももちろん可能です。
6. 日本語文献の `言語` フィールドに「ja」を設定します。
   - `著者名` フィールドを **右クリック（二本指クリック）** し、`Add Variant` → `Romaji` を選択して著者名のローマ字読みを登録すると、ローマ字順でソートできます。
7. 文献レコードをまとめて選択し、右クリックから `選択されたアイテムから参考文献目録を作成` を選びます。
   - `引用スタイル` に `Juris-Mで歴史学（日本語）`、`出力モード` に `参考文献目録`、`出力方法` に `クリップボードにコピー` を選択し、`OK` をクリックします。
   - テキストエディタなどにコピーし、文献一覧の表示を確かめます。
   - 文献中の引用表記を確かめたい場合は、`出力モード` で `出典表記` を選べば確認できます。



## プラグインとの連携 <a name="plugin"></a>

ZoteroやJuris-Mが威力を発揮するのはワープロソフトと連携したときです。対応しているワープロソフトは以下のとおりです。

- **Microsoft Word**：Zoteroのインストール時に自動的にアドインが追加されます。
- **Google Docs**：ブラウザ拡張機能（Zotero Connector）をインストールすると自動的に連携されます。Chrome・Firefox・Edge・Safariに対応しています。
- **LibreOffice**：Zoteroのインストール時に自動的にアドインが追加されます。

アドインが見当たらない場合は、`設定`（Juris-Mでは`環境設定`）>`引用`>`ワード・プロセッサ` から手動でインストールできます。

以下はWordを例に、文献情報の挿入手順を説明します。

1. Wordを開き、ツールバーに `Zotero` タブが用意されていることを確かめます。
2. Wordで文献情報を挿入してみます。
   - ツールバーから `Zotero`>`Add/Edit Citation` と選びます。
   - ポップアップ検索ウィンドウが出てくるので、著者名を入力すると該当する文献がサジェストされるので選択します。「佐藤, 2020」のように水色の四角で囲まれた文献項目が表示されます。Enterを押せば文献情報が挿入されます。
   - (佐藤 2020)のように文末引用がデフォルトなので、佐藤(2020)のように文中表記したい場合は、Enterを押す前に水色の文献項目をクリックし、`著者名の非表示` を選びます。
   - ページ数を記載する場合も同様に、Enterを押す前に文献項目のオプションを開き、`ページ数` に入力してください。

このリポジトリにWordのサンプルファイル sample.docx もあるのでご自由にお試しください（jss-lewis-sato.cslおよびJuris-M向けです）。



## 課題 <a name="todo"></a>

- Zoteroで使用する場合、文献一覧は日本語文字コード順にソートされます。ローマ字読みによるソートにはJuris-Mが必要です。
- 文献種別ごとの出力形式は主要な歴史学雑誌を参考に設定していますが、投稿先ごとに細部が異なる場合があります。
